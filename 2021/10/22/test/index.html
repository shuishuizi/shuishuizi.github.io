4.hexo安装、生成项目

4.1.安装hexo

cnpm install -g hexo-cli安装
hexo -v检测有没有安装成功，出现版本号之类的说明成功了

4.2.初始化一个hexo项目 (git bash管理员模式下)

创建一个空文件夹存放hexo项目，这个文件夹必须为空才能创建hexo项目在里面

用管理模式的git bash进入这个文件夹
输入hexo init初始化一个hexo项目
如果报错fatal: unable to access 'https://github.com/hexojs/hexo-starter.git/': OpenSSL SSL_read: Connection was reset, errno 10054，就继续输入hexo init，这是因为连接github是国外网站，容易连接不稳定、连接超时，直到出现下面这种就是创建hexo项目成功了

4.3.启动hexo (git bash管理员模式下)

hexo s或hexo server # 启动服务
这时就可以通过http://localhost:4000访问本地的博客界面，如果出现了界面就说明hexo项目已经构造完成，之后就可以根据自己的需求优化和上传到仓库了

5.上传hexo项目到github仓库

5.1.仓库名是固定格式

固定格式：用户名.github.io

5.2.安装一个插件：hexo-deployer-git自动部署发布工具 (git bash管理员模式下)

npm install hexo-deployer-git --save

5.3.配置_config.yml文件

5.4.生成静态文件
